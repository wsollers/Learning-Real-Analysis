# Structure Rules

Use this module for repository, volume, chapter, index, stub, capstone,
and file-layout checks.  Do not load these rules for isolated definition
or theorem body generation unless a generated artifact touches file
placement or navigation.

Primary covered areas:

- top-level repository layout
- volume scope
- chapter registry
- canonical chapter structure
- stub chapter requirements
- chapter opening pattern
- breadcrumb rules
- capstone placement
- input-chain rules

Machine-readable rule objects live in `design-rules.json` with
`module: "structure"`.
