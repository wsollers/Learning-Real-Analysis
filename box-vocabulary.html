Standard Remark Sequence

Not every remark applies to every result. Use what is warranted. Order matters â€” go from big-picture to technical.

After any lemma, proposition, definition, or theorem

  • [Fully quantified form] â€” The statement with all quantifiers written out explicitly: âˆ€n âˆˆ â„•, âˆ€m âˆˆ â„•, â€¦

The Governing Rule â€” Draft 1

See DESIGN.md in the project root for the full rules in text form with LaTeX templates.

Rule R1 â€” What gets a box

  • DEF Every definition gets a box. A definition is any statement of the form "X is [defined as / called / denoted] Y." This includes recursive definitions, notational conventions introduced for the first time, and official operator definitions (addition, multiplication, order). Never box a definition a second time when you use it â€” just cite it by number.
  • AX Every axiom gets a box. An axiom is an unproven primitive: Peano axioms, ZFC axioms, field axioms, vector space axioms. When an entire axiom system is listed (e.g. all eight vector space axioms), one box holds the full list. The axiom box is the most visually distinct â€” violet â€” because axioms are where mathematics makes its foundational bets.
  • THM Major named theorems get a box â€” but only on first appearance. A "major named theorem" is one that: (a) has a proper name (Compactness Theorem, De Morgan's Laws, Heine-Borel), or (b) is the main result of a section and will be cited by name throughout the rest of the notes. Lemmas, propositions, and corollaries are never boxed â€” they flow as plain amsthm.
  • KIT Each section gets at most one Toolkit box. The toolkit (gray) is only for the quick-reference summary table at the opening of a section. One per section, placed before the bare lemmas/propositions begin. Never mid-section.

Rule R2 â€” What never gets a box

  • Lemmas: always bare. A lemma is infrastructure, not a destination.
  • Propositions: always bare. Same reasoning â€” they derive from definitions and lemmas.
  • Corollaries: always bare. They are consequences, not standalone landmarks.
  • Remarks: always bare. If a remark needs emphasis, use italic text or a bold sentence opener, not a box.
  • Examples: always bare.
  • Proofs: always bare.
  • Second appearances: if Definition 2.2.1 was boxed in Â§2.2, never box it again in Â§3.4 when you reference it. Just cite: "By Def. 2.2.1 â€¦"

Rule R3 â€” The "Import" test (proposed heuristic)

  • Ask: "Is this the statement that someone opening to this section needs to immediately understand and remember?" If yes â†’ box it. If it's something that follows from the boxed material â†’ don't box it.
  • Concretely: In a section on Vector Spaces, the definition of a vector space is boxed. The proof that the zero vector is unique is not boxed. If the section is specifically about the Hahn-Banach Theorem and that theorem is the entire reason the section exists, it gets a theorem box.
  • Anti-pattern to avoid: Boxing every theorem in notes-equivalences.tex. Currently that file has 12 \begin{theorem} entries for things like Double Negation, Commutativity, Associativity. These are all consequences of the semantic definition; they should be bare propositions or collected into a single Toolkit table.
