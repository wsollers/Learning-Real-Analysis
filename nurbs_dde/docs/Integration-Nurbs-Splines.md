# NURBS_DDE Mathematical Laboratory: Integration, Approximation, Splines, NURBS, Frames, and Motion

## 1. Mission Statement

The goal of this document is to define a unified implementation roadmap for a full-featured mathematical laboratory inside `nurbs_dde`.

The laboratory should support two major workbenches:

1. **Integration Simulation and Approximation Laboratory**
2. **Spline, NURBS, Frame, and Motion Laboratory**

These should not be implemented as unrelated mini-apps. They should share a common mathematical workspace kernel built around:

```text
Geometry Objects
Parameter Domains
Sampling / Partitioning
Differential Queries
Integration / Approximation
Motion / Dynamics
Visualization / Diagnostics
Metadata / Units / Result Traces
```

The integration lab teaches accumulation over domains.

The spline/NURBS lab teaches geometry generated by parameterized objects.

Together, they form the basis for computational differential geometry, geometric simulation, curve/surface analysis, and interactive mathematical experimentation.

---

## 2. High-Level Design Principle

The core abstraction should be:

```text
Mathematical Object
+ Domain
+ Sampling / Partition
+ Differential Data
+ Numerical Method
+ Visualization
+ Diagnostics
```

The same architecture should support:

```text
1D integration
2D integration
3D integration
arc length
surface area
volumes of revolution
line integrals
surface integrals
NURBS curve analysis
NURBS surface analysis
Frenet frame motion
Bishop frame motion
swept geometry
particle motion along curves
curvature and torsion diagnostics
```

Avoid hardcoding the app around "rectangles under a graph."

Instead, treat integration generally as:

\[
\text{Integral} \approx \sum_i \text{sample value}_i \cdot \text{cell measure}_i.
\]

Then:

```text
1D integral:          cell measure = dx
arc length:           cell measure = ds
area integral:         cell measure = dA
surface integral:      cell measure = dS
volume integral:       cell measure = dV
Stieltjes integral:    cell measure = dg
curve mass:            cell measure = rho ds
surface mass:          cell measure = rho dS
```

## 5. Integration Problem Model

```text
Integral Problem
=
```

\[
\int_D f \, d\mu
\]

```text
D      domain

```

\[
\int_a^b f(x)\,dx
\]

## 6. Integration Modes

### 6.1 Tagged Riemann Integral

Support general tagged partitions:

\[
P = \{x_0, x_1, \dots, x_n\}
\]

with tags:

\[
x_i^\* \in [x_{i-1},x_i].
\]

Approximation:

\[
\sum_{i=1}^{n} f(x_i^\*)\Delta x_i.
\]

Required features:

```text
left tags
right tags
midpoint tags
random tags
custom draggable tags
arbitrary nonuniform partitions
live contribution display
```

### 6.2 Darboux Integral

Support upper and lower sums:

\[
U(f,P)=\sum_i M_i\Delta x_i
\]

\[
L(f,P)=\sum_i m_i\Delta x_i
\]

where:

\[
M_i=\sup_{x\in[x_{i-1},x_i]} f(x)
\]

\[
m_i=\inf_{x\in[x_{i-1},x_i]} f(x)
\]

The main diagnostic:

\[
U(f,P)-L(f,P).
\]

Required features:

```text
upper rectangles
lower rectangles
oscillation per cell
U-L gap visualization
Darboux integrability microscope
refinement animation
discontinuous examples
```

### 6.3 Riemann-Stieltjes Integral

Support:

\[
\int_a^b f(x)\,dg(x).
\]

Approximation:

\[
\sum_i f(x_i^\*)\left(g(x_i)-g(x_{i-1})\right).
\]

Required features:

```text
integrator function g
jump functions
step functions
distribution functions
mixed discrete/continuous accumulation
weighted accumulation visualization
probability expectation mode
```

This is a bridge toward measure theory.

### 6.4 Improper Riemann Integrals

Support improper integrals as limits:

\[
\int_a^\infty f(x)\,dx
=
\lim_{b\to\infty}\int_a^b f(x)\,dx.
\]

\[
\int_a^b f(x)\,dx
=
\lim_{\epsilon\to0^+}\int_{a+\epsilon}^b f(x)\,dx
\]

for singularities at endpoints.

Required features:

```text
infinite interval truncation
singularity exclusion window
limit convergence monitor
principal value mode
divergence detector
```

## 22. Curve Differential Quantities

For a parametric curve:

\[
\gamma(u)
\]

First derivative:

\[
\gamma'(u)
\]

Second derivative:

\[
\gamma''(u)
\]

Third derivative:

\[
\gamma'''(u)
\]

Speed with respect to parameter:

\[
\|\gamma'(u)\|
\]

Unit tangent:

\[
T(u)=\frac{\gamma'(u)}{\|\gamma'(u)\|}.
\]

Curvature in 3D:

\[
\kappa(u)
=
\frac{\|\gamma'(u)\times\gamma''(u)\|}
{\|\gamma'(u)\|^3}.
\]

Torsion:

\[
\tau(u)
=
\frac{(\gamma'(u)\times\gamma''(u))\cdot\gamma'''(u)}
{\|\gamma'(u)\times\gamma''(u)\|^2}.
\]

---

## 23. Frenet Frame

The Frenet frame is:

```text
T = unit tangent
N = principal normal
B = binormal
```

\[
T = \frac{\gamma'}{\|\gamma'\|}
\]

\[
N = \frac{T'}{\|T'\|}
\]

\[
B = T \times N
\]

The lab should display this frame moving along the curve.

### 23.1 Frenet Frame Failure Modes

The Frenet frame fails or becomes unstable when:

```text
||gamma'(u)|| ≈ 0
curvature ≈ 0
gamma' × gamma'' ≈ 0
torsion denominator ≈ 0
curve has inflection point
curve has nearly straight segment
numerical derivative noise dominates
```

When detected, the UI should show warnings.

---

## 26. Motion Along Curves

A particle moves along a curve by:

\[
p(t)=\gamma(u(t)).
\]

Velocity:

\[
\dot{p}(t)=\gamma'(u(t))u'(t).
\]

Physical speed:

\[
v(t)=\|\dot{p}(t)\|.
\]

Acceleration:

\[
\ddot{p}(t)
=
\gamma''(u(t))(u'(t))^2+\gamma'(u(t))u''(t).
\]

For arc-length motion:

\[
\ddot{\gamma}=v'T+\kappa v^2N.
\]

This decomposition should be displayed visually:

```text
velocity vector
acceleration vector
tangential acceleration
normal acceleration
curvature-driven acceleration
```

## 29. Continuity Laboratory

The continuity lab should inspect joins between curve segments.

### 29.1 Parametric Continuity

\[
C^0, C^1, C^2, C^3,\dots
\]

A join is \(C^k\) if derivatives agree up to order \(k\):

\[
\gamma_1^{(j)}(b)=\gamma_2^{(j)}(a)
\quad
\text{for } 0\le j\le k.
\]

### 29.2 Geometric Continuity

\[
G^0, G^1, G^2,\dots
\]

Interpretation:

```text
G0: endpoints meet
G1: tangent directions agree
G2: curvature agrees
G3: curvature variation agrees
```

A curve can be visually smooth under \(G^1\) while failing \(C^1\) because derivative magnitudes differ.

### 29.3 Derivative Agreement Diagnostics

Use \(D^k\) as a raw derivative-matching diagnostic:

\[
D^k\text{-agreement}:
\gamma_1^{(k)}(b)=\gamma_2^{(k)}(a).
\]

Required report:

```text
C0 pass/fail
C1 pass/fail
C2 pass/fail
G0 pass/fail
G1 pass/fail
G2 pass/fail
D1 jump magnitude
D2 jump magnitude
tangent angle mismatch
curvature jump
torsion jump
expected frame jump
expected acceleration jump
```

## 34. Arc-Length Reparameterization Service

## 35. Surface / NURBS Surface Laboratory

For a NURBS surface:

\[
S(u,v)
=
\frac{\sum_i\sum_j N_{i,p}(u)M_{j,q}(v)w_{ij}P_{ij}}
{\sum_i\sum_j N_{i,p}(u)M_{j,q}(v)w_{ij}}.
\]

Required features:

```text
control net
u-isocurves
v-isocurves
normal field
surface tangent vectors
first fundamental form
second fundamental form
Gaussian curvature
mean curvature
principal directions
parameter distortion heatmap
area element heatmap
surface integration support
```

### 35.1 Surface Differential Quantities

\[
S_u = \frac{\partial S}{\partial u}
\]

\[
S_v = \frac{\partial S}{\partial v}
\]

Normal:

\[
n = \frac{S_u \times S_v}{\|S_u \times S_v\|}.
\]

Area element:

\[
dS=\|S_u\times S_v\|\,du\,dv.
\]

This directly connects to the integration lab.

## 49. Important Design Warnings

### 49.1 Do Not Couple Rendering to Math

Rendering should consume sampled data, not own the mathematical object.

### 49.2 Do Not Couple Integration to 1D Graphs

Rectangles under curves are one visualization of a general integration process.

### 49.3 Do Not Treat Splines as Pure Graphics

Splines must be differentiable geometric objects with derivatives, curvature, torsion, and arc length.

### 49.4 Do Not Assume Frenet Frames Always Exist

Frenet frames fail near zero curvature. Bishop frames are required.

### 49.5 Do Not Hide Parameterization

Parameterization is central. Always distinguish:

```text
parameter u
time t
arc length s
physical speed
parameter speed
```

### 49.6 Do Not Ignore Numerical Stability

Every method should expose error, stability, and conditioning when possible.
